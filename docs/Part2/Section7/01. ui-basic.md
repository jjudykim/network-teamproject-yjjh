### 🔹 UI 생성 시 구조

---

- 하이어라키에서 `우클릭 > UI > Button (Legacy)` 선택 시 다음 구성 요소가 자동 생성됨
    - **Canvas**: UI 전용 도화지 역할
    - **Button**: UI 컴포넌트
    - **Text (Text - Legacy)**: 버튼에 들어가는 기본 라벨
    - **EventSystem**: UI 입력 이벤트 처리용 매니저

<br>
<br>

### 🔹 Canvas란?

---

- **UI 전용 공간 (2D 공간)**
- UI는 무조건 **Canvas 내부에 배치**되어야 렌더링됨
- 3D 오브젝트처럼 빈 오브젝트에 UI 컴포넌트를 붙여도 Canvas가 없으면 화면에 출력되지 않음
- 하나의 Canvas에는 **여러 개의 UI 오브젝트 추가 가능**
- UI 계층은 **부모-자식 관계**로 관리되며, 배치 순서도 영향을 미침

<br>
<br>

### 🔹 RectTransform의 개념

---

- UI 요소는 일반 `Transform`이 아닌 `RectTransform`을 사용
- RectTransform은 다음 기능을 담당함
    - 위치 (Position)
    - 크기 (SizeDelta)
    - 회전 (Rotation)
    - 피벗(Pivot)
    - 앵커(Anchor)

<br>
<br>

### 🔹 UI 편집 모드 & 단축키

---

- 씬 뷰 상단 `2D 버튼` 클릭 → UI 2D 모드로 전환
- 툴 조작 단축키
    - `Q`: Hand (뷰 이동)
    - `W`: Move
    - `E`: Rotate
    - `R`: Scale
    - ⭐`T`: Rect Tool (이동+회전+스케일 통합 조작)
- **Rect Tool 모드에서**
    - 드래그 → 위치 이동
    - 외곽 마우스 → 크기 조절
    - 모서리 마우스 (커서가 🔄) → 회전 가능
    - `Shift + Drag` → 비율 유지하면서 스케일 조절

<br>

### 🔹 Pivot (피벗)이란?

---

![image.png](attachment:934a9121-2ebd-4d75-a94a-fa7e65e40cc7:image.png)

- **UI 오브젝트의 중심점 (가운데 파란 동그라미!** 🔘)
- 회전, 스케일 등의 기준이 되는 점
- 직접 드래그하거나 수치 입력으로 조정 가능
- 예: Pivot이 중앙이면 정중앙 회전, 왼쪽 아래면 그 점을 기준으로 회전

<aside>
💡

**Global vs Local**

- Global: 씬 전체 기준의 UI 위치
- Local: 해당 오브젝트 기준의 위치 (로컬 좌표계)
- 대부분의 UI 편집은 **Local 기준으로 작업**
</aside>

### 🔹 Anchor (앵커)란?

---

![image.png](attachment:934a9121-2ebd-4d75-a94a-fa7e65e40cc7:image.png)

- 가운데에 모여있는 4개의 모서리 핀셋! (대충 이런 느낌으로 생긴.. 💠)
- **UI 요소가 부모(RectTransform) 기준으로 어디에 고정될지를 정하는 설정**
- 캔버스나 패널의 크기가 바뀔 때, **UI의 위치와 크기를 어떻게 유지할지**를 결정함
- 예: 화면 우측 상단 고정 버튼 = 앵커를 우측 상단에 설정
- 드래그 또는 프리셋으로 설정 가능

### 🔹 UI가 카메라 원근법에 영향을 받지 않는 이유?

---

- UI는 World 좌표계가 아닌 **스크린 공간(Screen Space)**에서 렌더링됨
- UI는 **항상 같은 크기**로 보이며, 카메라 위치와 무관
- 3D 오브젝트는 **원근 투영**에 따라 멀면 작게, 가까우면 크게 보이지만,

  UI는 **2D 평면상의 고정된 위치와 크기**를 가지며, 해상도에 따라 상대적으로 배치됨


**UI 좌표 실험 요약**

- 버튼을 복제 후 Z값(깊이)을 변경해도 **크기는 동일**함
- 원근법 영향 없음 → UI 전용 렌더링 시스템으로 처리됨

⇒ 따라서 **해상도/화면비율에 따라 자동 조정되도록 Anchoring이 매우 중요**

<br>
<br>

### ✅ 결론 및 정리

---

UI 시스템은 단순히 "보여주는 요소"가 아니라,

- **Canvas → RectTransform → Anchoring** 구조를 기반으로
- **스크린 해상도 대응**, **정밀 배치**, **입력 처리**까지 가능한 **전용 시스템**

따라서 UI 개발을 할 때는 단순한 GameObject 조작이 아니라 **UI 전용 툴셋과 구조**를 기반으로 접근해야 함!
