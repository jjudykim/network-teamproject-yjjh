### 🔹 **Collision(충돌)의 개념**

---

- **물리적 충돌**뿐만 아니라 "객체 간의 겹침", "접촉" 등 **공간상의 접촉 전반**을 의미
- 게임 내에서는 **이벤트 트리거, 피해 판정, 입장 조건 등** 다양한 용도로 사용됨
- 충돌 처리는 성능에 직접 영향을 주므로 **간략화된 계산 구조가 필요**

**충돌 환경 예시 구성**

1. **Plane** 오브젝트 생성
    - 바닥 역할
    - `Transform > Reset`
    - `Scale`을 `(5, 5, 5)`로 확장
    - 머테리얼 추가하여 색상 변경 (예: 초록색 `Green.mat`)
    - 머테리얼은 드래그 드롭으로 Plane에 적용
2. **유니티짱 프리팹 배치**
    - 공중에 위치시킴
    - 중력 작용 테스트를 위해 위로 올려 놓음

### 🔹 **물리 적용을 위한 필수 컴포넌트 – Rigidbody**

---

- **Rigidbody**를 추가해야만 물리 엔진의 영향을 받음
- 유니티에서 Rigidbody는 **물리 시뮬레이션을 적용하는 핵심 컴포넌트**

✅ 주요 옵션

| 항목 | 설명 |
| --- | --- |
| `Mass` | 질량 (kg 단위) |
| `Use Gravity` | 중력 적용 여부 |
| `Is Kinematic` | 수동 이동 시 true로 설정 |
| `Constraints` | 회전/이동 제한 가능 |

```csharp
// Rigidbody 없이 중력 작동 ❌
// Rigidbody 추가 후 Use Gravity = true → 중력 적용 O
```

### 🔹 **충돌이 일어나지 않는 이유 – Collider 누락**

---

- Rigidbody만 있으면 중력은 적용되지만 **충돌 판정은 없음**
- **Collider 컴포넌트**가 양쪽 오브젝트에 있어야 충돌 발생

💡 **Unity 공식 문서 요약**

> Rigidbody는 물리 시뮬레이션을 위한 것이고, Collider는 충돌 범위를 정의함. 두 요소가 결합되어야 실제 충돌 동작이 수행됨.
>

- Plane에는 기본적으로 Collider 포함
    - Plane에는 `Mesh Collider`가 기본 포함되어 있음
    - 유니티짱에도 **별도로 Collider를 추가**해야 충돌 가능
- **Collider 종류와 선택**

  : 3D에서는 대표적으로 다음과 같은 Collider가 존재

  | Collider 종류 | 설명 |
      | --- | --- |
  | `Box Collider` | 직육면체 영역 |
  | `Sphere Collider` | 구형 충돌 영역 |
  | `Capsule Collider` | 반구+원기둥 형태, 캐릭터에 적합 |
  | `Mesh Collider` | 모델 형태 그대로 충돌 (비용 ↑) |

  ⇒ 유니티짱에 적용: `Capsule Collider`

### 🔹 **Edit Collider 기능**

---

- Collider 추가 후 → `Edit Collider` 클릭
- 마우스로 크기/위치 조정 가능
- 시각적으로 충돌 범위를 확인하면서 조절

**왜 단순 Collider를 쓰는가?**

- 실제 모델은 수많은 삼각형(메시)으로 구성되어 있어 **정확한 충돌 계산은 매우 비용이 큼**
- 대신 *단순 도형(Capsule, Box 등)*으로 대략적인 외형을 감싸서 충돌 판정 수행
- 장르에 따라 충돌 정밀도 조절
    - RPG: 대략적인 Collider OK
    - FPS: 머리/팔/다리 등 부위별 Collider 사용

<br>
<br>

### 🔹 **Collider는 양쪽에 필요하다**

---

> “한 손만 있어선 박수를 칠 수 없다.”
>
- 충돌을 위해서는 **두 오브젝트 모두** Collider를 가져야 함
- 유니티짱(Rigidbody + Collider) vs Plane(Collider) → 충돌 발생

⚠️ Plane의 `Mesh Collider`를 비활성화하면 → 충돌 없이 **유니티짱이 계속 낙하**함

<br>
<br>

### ✅ **결론 및 정리**

---

| 구성 요소 | 역할 | 필요 여부 |
| --- | --- | --- |
| Rigidbody | 물리 적용 (중력 등) | 유니티짱 O |
| Collider | 충돌 범위 정의 | 유니티짱 & Plane 모두 O |
| Material | 시각적 표현 (Optional) | 선택사항 |
| Shader | Material 구성 요소 | 선택사항 |
| Mesh | 시각적 모델 | 충돌과 무관 |
- 충돌 처리를 위해서는 **RigidBody + Collider의 조합**이 필수
- Collider는 **정확도와 성능의 균형**을 고려하여 선택
- 충돌은 게임에서 매우 빈번한 요소이며, 시스템 전반에 영향을 주므로 **철저한 구조 설계**가 필요
