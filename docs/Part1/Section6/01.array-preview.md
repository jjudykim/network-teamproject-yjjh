### 🔹 **배열 (Array)**

---

**개념**

- 동일한 자료형의 데이터를 연속된 공간에 저장하는 자료구조.
- 배열은 크기를 지정해서 한 번에 여러 데이터를 묶어서 관리할 수 있음.

**선언 방식**

```csharp
int[] scores = new int[5];  // 길이 5의 int형 배열
```

**데이터 입력**

```csharp
scores[0] = 10;
scores[1] = 20;
scores[2] = 30;
scores[3] = 40;
scores[4] = 50;
```

**초기화 단축 문법**

```csharp
int[] scores = new int[] { 10, 20, 30, 40, 50 };  // 배열 크기 생략 가능
int[] scores = { 10, 20, 30, 40, 50 };            // 가장 축약된 표현
```

**배열 접근**

- 인덱스는 0부터 시작함
- `scores[0]` 은 첫 번째 값
- 범위를 벗어난 인덱스 접근 시 에러 발생 → `IndexOutOfRangeException`

**배열 순회 방식**

```csharp
// 일반 for문
for (int i = 0; i < scores.Length; ++i)
    Console.WriteLine(scores[i]);

// foreach문 (가독성 ↑, 인덱스 필요 없는 경우에 좋음)
foreach (int score in scores)
    Console.WriteLine(score);
```

**배열의 길이**

- `scores.Length` : 배열의 전체 요소 개수 반환

**배열은 참조 타입**

- 배열을 다른 변수에 대입하면 주소를 공유함

```csharp
csharp
복사편집
int[] scores2 = scores;
scores2[0] = 99999;  // scores[0]도 99999로 변경됨

```

**배열은 동적 생성 후 변경 불가**

- `new int[5]` 로 생성하면 크기 5로 고정됨
- 이후 요소 수 변경 불가능 (다시 생성 필요)

### ✅ 결론

---

- 배열은 가장 기본적인 형태의 자료구조이며, 동일한 타입의 데이터를 일괄적으로 다룰 수 있게 해 줌
- `for` / `foreach` 루프를 통해 순회하며 데이터를 조회/조작 가능
- 배열은 참조 타입이기 때문에 주소 복사에 유의해야 함
- 범위 초과 접근은 치명적인 에러 (서버 다운 등의 원인)가 될 수 있음
- 이후 List나 Dictionary 등 더 유연한 컬렉션을 배우게 될 예정
