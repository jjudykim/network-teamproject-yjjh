### 🔹 **절차지향 vs 객체지향**

---

| **구분** | **절차지향 (Procedural)** | **객체지향 (OOP: Object-Oriented Programming)** |
| --- | --- | --- |
| **개념** | 함수 기반으로 코드를 설계 | 객체 단위로 코드를 설계 |
| **구성 요소** | 함수 (`Function`) 위주 | 클래스 (`Class`), 객체 (`Object`) 위주 |
| **데이터 관리** | 전역 변수나 구조체 활용 | 속성 (`Properties`)과 기능 (`Methods`)을 클래스 내부에서 관리 |
| **코드 유지보수** | 기능이 많아질수록 복잡해짐 | 재사용성과 유지보수가 용이 |
| **예제** | `CreatePlayer() → EnterGame() → Fight()` (순서 중요) | `Knight 객체 생성 → 행동 수행 (Move(), Attack())` |


<br>
<br>

### 🔹 **객체지향 프로그래밍 (OOP)**

---

- OOP는 객체 중심의 프로그래밍 방식
- 프로그램의 모든 것을 **객체**로 보고 속성과 기능을 정의하여 코드 재사용성을 높임

✅ **객체의 구성 요소**

- **속성 (Attributes) → 데이터**
- **기능 (Methods) → 행동**

<br>
<br>

### 🔹 **클래스 (Class)**

---

```csharp
class Knight
{
    public int hp;
    public int attack;

    public void Move()
    {
        Console.WriteLine("Knight Move");
    }

    public void Attack()
    {
        Console.WriteLine("Knight Attack");
    }
}
```

✅ **클래스 문법**

- `class` 키워드를 사용하여 객체의 설계도를 정의
- `public int hp;` → **속성 (Attributes)**
- `public void Move() {}` → **기능 (Methods)**


<br>
<br>

### 🔹 **객체 생성 (`new` 키워드)**

---

```csharp
static void Main(string[] args)
{
    Knight knight = new Knight();  // 객체 생성
    knight.hp = 100;
    knight.attack = 10;

    knight.Move();                 // "Knight Move" 출력
    knight.Attack();               // "Knight Attack" 출력
}
```

✅ **객체 생성 원리**

- `new Knight();` → 클래스의 인스턴스를 생성
- `knight.hp = 100;` → 객체 속성에 값을 설정
- `knight.Move();` → 객체의 기능을 호출


<br>
<br>

### 🔹 **객체 없이 접근 시 (`null` 오류)**

---

```csharp
Knight knight = null;
knight.Move();  // ❌ Null Reference Exception 발생
```

✅ **오류 발생 이유**

- `null` 상태에서는 객체가 메모리에 존재하지 않음
- `Move()` 같은 기능을 호출하면 **런타임 오류 (Null Reference Exception)** 발생
- **해결 방법**: 반드시 `new`를 사용하여 객체를 생성해야 함


<br>
<br>

### 🔹 **객체지향 프로그래밍의 핵심**

---

**✅ 1. 캡슐화 (Encapsulation)**

- 객체 내부의 **데이터 보호** (예: `private` 키워드 활용)

**✅ 2. 상속 (Inheritance)**

- 기존 클래스를 확장하여 새로운 클래스를 정의 (예: `class Mage : Knight {}`)

**✅ 3. 다형성 (Polymorphism)**

- 같은 기능 (`Attack()`)을 여러 방식으로 다르게 구현 가능

<br>
<br>

### 🔹 **마무리**

---

- **클래스**: 객체의 설계도
- **객체**: `new`를 통해 클래스를 기반으로 생성된 인스턴스
- **속성 (Attributes)**: 데이터 (예: `hp`, `attack`)
- **기능 (Methods)**: 행동 (예: `Move()`, `Attack()`)
